# PostMON

Minimal desktop UI to run Postman collections without opening the CLI.

Under the hood it runs **Newman** (the official Postman collection runner) with the **htmlextra** reporter. PostMON adds folder selection, UI languages, sensible defaults and an extra HTML report with full request/response data.

## Download and run (Windows)

1. Open the [Releases page](../../releases) and pick the latest version
2. Download one of the assets:
   - **`PostMON-<version>-win-unpacked.zip`** — extract anywhere, run `PostMON.exe` from the extracted folder. No install
   - **`PostMON-<version>.exe`** — portable single executable, no install
3. Double-click `PostMON.exe`

### Windows SmartScreen warning

The build is **not code-signed** (signing a Windows app costs ~€200/year). First launch Windows may show **"Windows protected your PC"**. To run anyway:

1. Click **More info**
2. Click **Run anyway**

The source is open and the build is produced by [GitHub Actions](.github/workflows/release.yml) from this repo, so you can audit and rebuild it yourself.

### First run

- Click **Browse** to load a `.json` Postman collection, or **Folder** to scan a directory
- Pick zero or more folders inside the collection (empty = run all)
- Click **Run All** or **Run Selected**
- Reports land in `%APPDATA%\PostMON\reports\` by default, or in the folder you choose
- Click **Open report** to view the HTML, or **Save report...** to copy it elsewhere

## What it does

- load a `.json` collection exported from Postman, or scan a folder to find them
- pick single folders inside the collection or run the whole thing
- optional: environment, globals, proxy, request timeout, request delay (default 100 ms), SSL insecure
- run Newman and watch requests, status codes, times and test pass/fail in real time
- HTTP 4xx/5xx are not marked as failures if the collection's tests treat them as valid
- generate two reports:
  - **PostMON report** — compact layout with expandable request/response (headers and body) per call
  - **htmlextra report** — Newman's classic enhanced HTML reporter with full request/response data
- see the equivalent Newman CLI command generated by the UI
- 6 UI languages: Italiano, English, Français, 中文, 日本語, Русский (picker in the top-right header, choice is persisted)

Response bodies are capped at 200 KB in the PostMON report to keep the HTML manageable.

## How it works

PostMON is an Electron wrapper around the `newman` Node library (no shell calls). When you start a run:

1. If you picked folders, it builds a temporary collection containing only the selected branches, preserving parent collection scripts, auth and variables
2. It calls `newman.run()` with `json` + `htmlextra` reporters
3. It captures request/response data via runtime events and writes the extra PostMON report
4. Everything is saved to `reports/` inside the app userData directory, or to the folder you chose

Collections and reports stay on your filesystem. No telemetry, no network calls beyond those the collection itself makes.

## Development

## Security

- Electron with `contextIsolation: true`, `nodeIntegration: false`, typed IPC via preload
- Local app, no telemetry
- `.gitignore` excludes `*.postman_collection.json`, `*.postman_environment.json`, `*.postman_globals.json`, `reports/`, `dist/`, `.env*` to prevent token/credential leaks
- Never commit collections/environments with hardcoded secrets
